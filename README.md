# Four Horses Club — лендинг

## Подготовка перед версткой

Перед началом вёрстки лендинга выполнена **декомпозиция макета**:

### Структура проекта

Корень репозитория организован так:

- **`assets/`** — статика: изображения, иконки, шрифты
  - `assets/images/` — растровая и векторная графика лендинга
  - `assets/icons/` — иконки
  - `assets/fonts/` — файлы шрифтов
- **`css/`** — стили: сброс, переменные, базовые слои, компоненты, общий `styles.css`
- **`js/`** — скрипты (`main.js`)
- **`index.html`** — точка входа страницы

Такая схема позволяет сразу подключать ассеты предсказуемыми путями и не смешивать разметку, стили и медиа в одной куче.

### HTML-скелет и семантика

- Подготовлен базовый HTML-скелет страницы в `index.html` в соответствии с требованиями семантики.
- Использованы семантические блоки (`header`, `main`, `section`, `footer`) и корректная иерархия заголовков по секциям.
- Добавлены базовые практики доступности: осмысленные `alt`, `aria`-атрибуты по назначению, а также структура для анимируемых блоков без дублирования контента для скринридеров.

### Шрифты

- Вариативные шрифты приведены к **единым именам файлов** в `assets/fonts/` (без служебных суффиксов в духе `VariableFont_…`).
- Для веба созданы **WOFF2**-файлы (основной формат для `@font-face`).
- Подключение описано в **`css/fonts.css`**, агрегировано через **`css/styles.css`** (`@import "./fonts.css"`).

### Изображения

- Тяжёлые ассеты **сжаты** там, где размер выходил за разумные для веба границы.
- Для ряда картинок добавлены **WebP**-аналоги рядом с исходниками;

### CSS-переменные

- Подготовлен файл `css/variables.css` с цветовыми токенами для текста, акцентов и фонов.
- Добавлены переменные для состояний кнопок: primary, secondary, third и disabled.
- Добавлена переменная фона секций (`--color-bg-section`) для единообразного оформления блоков.

### BEM-классы в разметке

- HTML-разметка в `index.html` дополнена BEM-классами для основных блоков и элементов.
- Подготовлена структура классов для секций hero/about/plan/participants, карточек участников, тикера и кнопок с модификаторами.
- Это упрощает дальнейшую модульную стилизацию и поддержку проекта при масштабировании.
